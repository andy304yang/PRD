## 需求描述

前端接口变更说明：
POST /api/v1/agent/create 对话生成分身接口新增 5 轮识别上限，接口请求参数不变，仍使用：
{
  "userInput": "用户输入内容",
  "userConfirm": false
}
后端会按当前会话内普通用户输入轮次计算，不需要前端传 roundCount。userConfirm=true 只用于确认生成，不计入识别轮次。
当第 5 轮普通输入后，模型仍返回 canGenerateAgent=false 时，后端会终止当前对话会话，并返回统一错误：
{
  "success": false,
  "code": "BUSINESS_ERROR",
  "message": "暂时还不能生成分身",
  "data": null,
  "traceId": "...",
  "timestamp": "..."
}
前端收到该错误后，应禁止用户继续在当前对话里发送消息，并展示失败提示。用户如果再次发起对话，后端会自动开启新的对话会话，不会继续使用已失败会话的历史记录。第 5 轮如果 canGenerateAgent=true，接口行为保持不变，前端仍可展示【直接生成】等操作。
