## 需求描述

前端 SKILL 分身派遣功能 - 基于 SKILL 的分身任务派发机制：

1. **有 SKILL 才能派发分身**：用户必须至少拥有一个 SKILL 才能派出分身执行任务
2. **无 SKILL 时弹窗提示**：如果没有 SKILL，弹出提示框引导用户去新增 SKILL
3. **删除 SKILL 不影响执行中的任务**：假设用户有一个 SKILL 且有一个执行中的任务，删除该 SKILL 后：
   - 执行中的任务继续正常运行，不受影响
   - 但不能再派出新的分身任务
4. **SKILL 存在性检查**：每次派发任务前检查是否有可用的 SKILL，没有则弹窗

## 功能逻辑

### 分身派遣条件
- 前置条件：用户已创建并保存至少一个 SKILL
- 检查时机：点击"派出分身"按钮时

### 弹窗触发场景
| 场景 | 行为 |
|---|---|
| 无任何 SKILL | 弹出提示："请先创建一个 SKILL 才能派出分身" + 跳转新增按钮 |
| 有 SKILL 但全被删除 | 同上 |
| 有 SKILL 且有执行中任务 | 允许派发新任务 |

### SKILL 删除保护机制
- 删除 SKILL 时系统检查是否存在该 SKILL 关联的执行中任务
- 若有执行中任务：允许删除，但该 SKILL 相关元数据标记为"已删除-执行中"
- 执行中任务继续运行直至自然结束
- 新任务派发时不再可选用已删除的 SKILL

## 技术要求

- 前端状态管理需区分"已删除 SKILL"和"执行中任务"
- 删除操作应为软删除（标记位）而非硬删除
- 执行中任务需关联 SKILL ID，任务结束后再清理关联
