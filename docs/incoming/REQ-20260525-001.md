## 需求描述

搭建项目监控自动化链路：

- 飞书 @机器人 发需求 → Hermes 接收
- Hermes 生成标准 PRD Markdown → push 到 GitHub PRD 仓库
- Codex cron 监控 PRD 仓库 push → 发现新需求
- Codex 拉取 PRD → 理解需求 → 修改代码 → 验证
- Codex 调用 Harness 多 agent 测试 → 生成测试报告
- Codex 把报告上传 COS → Hermes 通知飞书

## 链路角色分工

| 角色 | 职责 |
|------|------|
| Hermes | 触发、整理、入库、通知 |
| Codex | 理解需求、改代码、验证、报告 |

## 技术方案

### Hermes（飞书端）

- 接收飞书机器人 @消息（飞书开放平台自建应用，App ID: cli_a960be88b3f9dcee）
- 生成标准 PRD 文档，格式：
  - docs/incoming/REQ-YYYYMMDD-NNN.md
  - 包含：需求 ID、背景、用户故事、目标页面、交互说明、验收标准
- push 到 GitHub：github.com/andy304yang/PRD
- 维护任务状态表

### Codex（自动化端）

- cron 自动化，每小时检查 PRD 仓库 push
- 发现新 PRD 后拉取代实现
- 按项目 skills（using-superpowers、qxun-demo 相关规则）实现功能
- 调用 Harness 多 agent 测试
- 输出测试报告，上传 COS

### 通信方式

- Hermes ↔ GitHub：git clone/add/commit/push
- Hermes ↔ 飞书：飞书开放平台 API
- Codex ↔ Hermes：Codex 回调 POST 到 Hermes API
- Hermes ↔ Codex：COS 作为中转（manifest + 报告文件）
